## The Eight Golden Rules

1. **Strive for consistency** - Consistent sequences of actions in similar situations. Identical terminology in prompts, menus, and help screens. Consistent color, layout, capitalization, fonts throughout.

2. **Seek universal usability** - Recognize the needs of diverse users. Accommodate novice to expert, age ranges, disabilities, and technological diversity. Add features for novices (explanations) and experts (shortcuts).

3. **Offer informative feedback** - For every user action, there should be interface feedback. Frequent and minor actions: modest feedback. Infrequent and major actions: more substantial feedback.

4. **Design dialogs to yield closure** - Sequences of actions should be organized into groups with a beginning, middle, and end. Informative feedback at the completion of a group gives satisfaction and preparation for the next group.

5. **Prevent errors** - Design the interface so that users cannot make serious errors. If a user makes an error, the interface should detect it and offer simple, constructive instructions for recovery. Erroneous actions should leave the system state unchanged, or give instructions about restoring the state.

6. **Permit easy reversal of actions** - As much as possible, actions should be reversible. This relieves anxiety and encourages exploration.

7. **Keep users in control** - Experienced users want to be in charge. Avoid surprising actions, tedious data-entry sequences, inability to obtain necessary information, or difficulty in getting the result they want.

8. **Reduce short-term memory load** - The limitation of human information processing in short-term memory (7 +/- 2 chunks) requires that displays be kept simple, multiple-page displays be consolidated, and sufficient training time be allotted.

## When to apply
- Complementary to Nielsen's heuristics - use both for comprehensive evaluation
- Particularly strong on error prevention, reversal, and memory load
- Apply when evaluating complex workflows and multi-step processes
