## Four Pillars of Information Architecture

1. **Organisation** - How content is grouped and categorised. Schemes include alphabetical, chronological, geographical, topical, task-based, audience-based.

2. **Labelling** - How content is named. Labels should be familiar, consistent, and descriptive. Avoid jargon. Test with users.

3. **Navigation** - How users move through content. Global nav, local nav, contextual nav, breadcrumbs, sitemaps.

4. **Search** - How users find specific content. Essential when content exceeds what browsing can handle (typically 50+ items).

## When Search is Essential

- More than 50 items in a list or database
- Users know what they want but not where it is
- Content is deeply nested (3+ levels)
- Multiple valid paths to the same content

## Progressive Disclosure

Present information progressively - show the minimum needed at each step:
1. Overview first (summaries, titles, thumbnails)
2. Details on demand (expand, click through, drill down)
3. Full content last (complete data, raw details)

## Common IA Violations
- Flat lists of 50+ items with no grouping or search
- Category labels that use internal jargon instead of user language
- Inconsistent navigation across pages (different menus on different pages)
- Important content buried 3+ clicks deep with no shortcuts
- No indication of content volume within categories
